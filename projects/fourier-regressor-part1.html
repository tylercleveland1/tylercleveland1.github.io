Series Basics

For the basis of our learning we considered using both the Taylor series and the Fourier series.

The Taylor series is a sum of polynomials, and is derived for a given function based on the $nth$ derivative of that function. The series for $sin(x)$ and $cos(x)$ are below:

With the assumption that the period will be $2\pi$, the Fourier series of $K$ terms is basically represented by the following form: $$\alpha_0+ \sum_{k=1}^K(\alpha_k\sin(kx) + \beta_k\cos(kx)).$$

We chose the Fourier series for this project based upon it's (what we observed to be) more consistent format, both for the purpose of representing the learned equation but more importantly a clearer optimization process for picking coefficients for the final series.

Practical Parameters

When actually performing the learning, the number of terms $K$ will be preset as a hyperparameter.

From hereon out, the word "weight" refers to the coefficients of the series.

From a hyperparameter $K$, we will be learning $2*K + 1$ weights for the series ($2 * K$ because there is a $sin$ and a $cos$ for each term, and $+1$ because there is one weight in front.

Optimization

We will use batch gradient descent to perform the optimization process. Ideally we need to find a loss function that is convex and therefore can be reliably optimized.

Below is a drawn example of the optimization process:

We will optimize with respect to each weight individually.

The green line represents the series function that contains the weight we are trying to optimize. The red line represents the underlying frequency backing that weight. For example, we could be optimizing the series $$\alpha_0+ \alpha_1\cos(x) + \beta_1\sin(x).$$ with respect to $\alpha_1$, and our underlying frequency (red line) would be $cos(x)$.

Given the two blue training points on the graph, the function clearly needs to increase near both the left and the right training points to reduce error.

However, this is not as simple as increasing the weight based on the total error. This is because the function is oscillating, and a positive change in the weight doesn't necessarily represent a positive change at the output of the series.

For example, on both training points we see that the error ($true - predicted$) is going to be positive. So, it makes intuitional sense that increasing the weight will increase the output. But as seen in the underlying frequency, the left point has a positive value whereas the right point has a negative output. This means that increasing the weight will decrease the error on the left point, but actually $increase$ the error on the right.

So, we can multiply each error by the position of the underlying frequency at that point to account for this. For the left training point this will not have an effect on the sign, but for the right training point this will reverse the sign of the error. Then, we change the weight based on that new value, and the error will decrease.

Estimating and Loss Function

Suppose we have a set of $N$ 1-dimensional observations $X$, comprised of $x_i \in X$.

Let $Y$ be a vector containing all true values, comprised of $y_i \in Y$.

The weights $W$ and terms $T$ will be defined as

$$W=[\alpha_0, \alpha_1, \beta_1, \alpha_2, \beta_2, ..., \alpha_k, \beta_k]$$$$T = [1, \sin(x_i), \cos(x_i), \sin(2*x_i), \cos(2*x_i), ..., \sin(k*x_i), \cos(k*x_i)]$$

for $K$ terms in the series.

For all $x_i \in X$ let $t_i$ be $T$ as defined above evaluated at $x_i$

Then, we can say our error $e$ and MSE for a training point $x_i \in X$ with respect to a given weight $w \in W$ is:

$$e = (y_i - \hat y_i) * t(x_i)$$$$MSE = \frac{1}{N} *\sum_{i=1}^N((y_i - \hat y_i) * t(x_i))^2$$

where the predicted value $\hat y_i = t_i \cdot W$ and $t \in T$ is the term corresponding to weight $w$

Example Optimization

A small example is useful to help illustrate the problem. For the purposes of this explanation, we will use a 1-dimensional dataset generated based on a simple linear function $f(x) = x$ and a small series in the following form (general form above with $K = 1$):

$$\alpha_0+ \alpha_1\sin(x) + \beta_1\cos(x).$$

So, we need to optimize the weights $\alpha_1, \beta_1$. Assume $\alpha_0$ is optimal at 0 for the purpose of this demo.

We will graph the above with the weights set to: $\alpha_0 = 0, \alpha_1 = 1, \beta_1=1$, on the range $[-1, 1]$

import matplotlib.pyplot as plt
import numpy as np
import random

num_points = 100

f_x = lambda x: x

y = [f_x(x) for x in X]
y_series = [series_x(x, 1, 1) for x in X]

#a = alpha_1, b = beta_1
def mse(X, y, a, b, term):
  global series_x
  tot = 0
  for _x, _y in zip(X, y):
    tot += ((series_x(_x, a, b) - _y) * term(_x)) ** 2
  
  return tot / float(len(X))

coef_range = np.array(range(-100, 100)) / 10

loss_range_a = [mse(X, y, _a, 1, np.sin) for _a in coef_range]
loss_range_b = [mse(X, y, 1, _b, np.cos) for _b in coef_range]

fig = plt.figure()
ax1 = fig.add_subplot(211)
ax1.set_ylabel('MSE')
ax1.set_title('Weight value')
ax1.plot(coef_range, loss_range_a, color="green", label='alpha_1')
ax1.plot(coef_range, loss_range_b, color="blue", label='beta_1')
handles, labels = ax1.get_legend_handles_labels()
ax1.legend(handles, labels)
plt.show();

print('minima for alpha_1', coef_range[np.argmin(loss_range_a)])
print('minima for beta_1', coef_range[np.argmin(loss_range_b)])
